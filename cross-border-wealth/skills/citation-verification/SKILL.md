# /citation-verification

Verifies legal citations for accuracy, currency, and provenance. The quality gate that prevents hallucinated or outdated citations from reaching final work product. Every citation gets a provenance tag so the user knows exactly what has been verified and what requires manual confirmation.

## Instructions

1. **Load practice profile.** Read `~/.claude/plugins/config/legal-engineering/cross-border-wealth/CLAUDE.md`.

2. **Identify citations to verify.** Either:
   - User provides specific citations to check
   - User provides a document; extract all legal citations from it
   - Verify citations generated by other skills in this session

3. **Provenance tagging system.** Every citation gets exactly ONE tag:

   | Tag | Meaning | Confidence |
   |---|---|---|
   | `[Westlaw]` | Verified against Westlaw data | High |
   | `[CourtListener]` | Verified against CourtListener / public court records | High |
   | `[EUR-Lex]` | Verified against EUR-Lex / CURIA | High |
   | `[司法院]` | Verified against Taiwan Judicial Yuan database | High |
   | `[legislation.gov]` | Verified against legislation.gov.uk / national gazette | High |
   | `[MCP-verified]` | Verified via available MCP data source | Medium-High |
   | `[user-provided]` | Citation provided by user; not independently verified | Medium |
   | `[model-knowledge]` | From model training data; REQUIRES INDEPENDENT VERIFICATION | Low |
   | `[unverified]` | Could not verify; treat as suspect | Very Low |

4. **Verification checks for each citation:**

   ### Case citations
   - Does the case exist? (correct parties, court, year, reporter)
   - Is the citation format correct for the jurisdiction?
   - Is the case still good law? (not overruled, reversed, or distinguished on point)
   - Has the case been cited approvingly by higher courts?
   - Are the propositions attributed to the case actually in the judgment?

   ### Statutory citations
   - Does the section/article exist in the cited act?
   - Is the version current? (check for amendments, repeals)
   - Is the act still in force?
   - Are there pending amendments that would change the analysis?

   ### Regulatory guidance citations
   - Is the guidance document current? (not superseded)
   - Is the issuing authority correct?
   - Has the guidance been withdrawn or updated?

5. **Overruling/distinguishing check:**
   - Search for subsequent cases that cite the referenced case
   - Identify any negative treatment (overruled, doubted, not followed, distinguished)
   - If negatively treated, assess whether the specific proposition relied on is affected

6. **Format validation per jurisdiction:**

   | Jurisdiction | Format Standard | Example |
   |---|---|---|
   | US Federal | Bluebook (21st ed.) | *Smith v. Jones*, 500 U.S. 100, 105 (2020) |
   | US State | Bluebook + local rules | *People v. Smith*, 50 Cal.4th 100 (2020) |
   | UK | OSCOLA (4th ed.) | *Smith v Jones* [2020] UKSC 1 [45] |
   | EU | CURIA / EUR-Lex | Case C-100/20, *Smith* [2021] ECLI:EU:C:2021:100 |
   | Taiwan | 司法院 format | 最高法院110年度台上字第1234號判決 |
   | Taiwan (constitutional) | 釋字 format | 司法院大法官釋字第803號解釋 |

7. **Produce verification report:**

   | # | Citation | Provenance | Status | Issue | Action Needed |
   |---|---|---|---|---|---|
   | 1 | *Smith v. Jones* (2020) | `[CourtListener]` | ✅ Verified | None | None |
   | 2 | 26 USC § 7701(a)(30) | `[legislation.gov]` | ✅ Verified | None | None |
   | 3 | *Doe v. Roe* (2019) | `[model-knowledge]` | ⚠️ Unconfirmed | Cannot independently verify | Manual check required |
   | 4 | Case C-200/18 | `[EUR-Lex]` | ❌ Error | Case number does not exist | Remove or correct |
   | 5 | 最高法院109年台上字999號 | `[司法院]` | ⚠️ Partially verified | Case exists; proposition unclear | Verify holding |
